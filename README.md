# トレーニング記録・分析アプリ

毎日のトレーニング記録を蓄積し、週間・月間の変化や傾向を分析するWebアプリです。
スマホでの利用を想定したモバイルファーストのUIで、ビルド不要・サーバー不要の静的ファイルだけで動きます。

## 主な機能

- **記録タブ** — 日付ごとに種目（筋トレ／有酸素／ボクシング／回数系／その他）、時間帯（朝・昼・夕方・夜）、コンディション（合計運動時間・疲労度・睡眠・痛み・体重・体脂肪率・歩数・感想）を記録。入力は自動保存。
  - **部位から種目を選ぶ** — 「🎯 鍛えたい部位から種目を選ぶ」で、大分類（上半身／下半身／体幹・腹／有酸素／ボクシング／回復・ケア）→ 中分類（胸・背中・肩・腕など）→ 小分類（大胸筋上部・下腹部など）と絞り込むと候補の種目が表示され、タップすると種目名と入力欄の種別が自動でセットされます。
  - 「今日の記録をまとめる」で、仕様どおりの形式（■トレーニング／■合計運動時間／…）のまとめを表示・コピーできます。末尾には現在の主な目標も記載されます。
  - **AIで評価** — まとめ画面の「🤖 AIで評価」を押すと、記録＋目標＋評価の指示（結論から簡潔に・回復面の注意・目標を踏まえた次回への提案など）をまとめた評価用プロンプトが自動でコピーされます。そのままClaudeまたはChatGPTを開くボタンが表示されるので、貼り付けて送信するだけでAIの評価を受けられます。APIキーや追加費用は不要です。
  - **種目を調べる** — 種目名の横の「🔍 調べる」ボタン、または登録済み種目の名前をタップすると、アプリ内蔵の種目解説データベース（約40種目・別名対応）から、その種目の「鍛えられる部位・概要・やり方・コツ」を正確に表示します。主要な筋トレ・有酸素種目には**動作イメージ写真（①開始姿勢→②動作後の2枚組）**も表示されます（出典：free-exercise-db／パブリックドメイン。写真がその種目の動作と一致することを確認したものだけ掲載）。近い別種目で代替表示することはありません。種目名がWikipediaの記事と完全一致する場合のみ、参考として画像とリンクを添えます（出典表示付き）。あわせて「やり方を動画で見る（YouTube）」「フォームの画像を見る（Google画像）」「Webで詳しく検索（Google）」へのリンクを表示します。内蔵解説はオフラインでも表示できます。
- **履歴タブ** — 月ごとの一覧。タップでその日のまとめを表示。記録のない日から直接記録を始められます。
- **分析タブ** — 週間分析（月〜日）と月間分析。トレーニング日数／休養日数／記録なし日数／セッション数／総運動時間／種目別集計／体重・体脂肪率の推移グラフ／疲労・睡眠・痛みの傾向を表示。前週・前月のデータがあれば比較、なければ「前週比較なし」と表示します。
- **データタブ** — プロフィール（身長・性別・生年月日・目標体重・目標体脂肪率）、現在の主な目標の管理（追加・タップで編集・削除。記録と一緒に保存されます）、JSONエクスポート／インポート、全データ削除。
  - プロフィールを設定すると、最新の記録から**年齢・BMI・基礎代謝の目安（Mifflin-St Jeor式）・最大心拍数の目安（220−年齢）・目標までの残り**を自動計算して表示します。BMIと目標までの残りは、1日のまとめと週間・月間分析にも表示されます。

## 設計方針（ChatGPTプロンプト仕様の反映）

- 日付・曜日は**日本時間（JST）**基準（端末のタイムゾーンに依存しません）
- 未入力の数値は**推測せず「記録なし」**として扱う
- 「記録なし」と「休養日」を区別して集計する
- 運動時間が未記録の日は総運動時間に含めず、その旨を注記する
- 疲労度7以上・痛みの記録がある期間には注意メッセージを表示する
- 記録の追加・修正・削除はいつでも可能で、週間・月間集計に自動反映される

## 使い方

### パソコンで開く

`index.html` をブラウザで直接開くだけで動きます。

### スマホで使う（推奨：GitHub Pages）

1. このリポジトリをGitHubにpushする
2. GitHubのリポジトリ設定 → **Pages** → Branchを `main` に設定して公開
3. スマホで `https://<ユーザー名>.github.io/<リポジトリ名>/training-log-analysis-app/` を開く
4. ブラウザのメニューから**「ホーム画面に追加」**すると、アプリのように全画面で起動できます（オフラインでも動作します）

## データの保存場所と注意

- 記録はブラウザの **localStorage** に保存されます（端末内で完結、外部送信なし）
- **端末やブラウザを変えるとデータは引き継がれません。** データタブから定期的にJSONエクスポートしてバックアップしてください
- 別端末へ移行する場合は、エクスポートしたJSONをインポートしてください
